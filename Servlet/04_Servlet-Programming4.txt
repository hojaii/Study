* Cookie
: 서버가 사용자의 컴퓨터에 설치하는 작은 기록 정보 파일, 웹사이트 기본설정, 임시데이터 저장,
페이지 개인화를 위한 식별정보 송수신 가능


* Session
: 상태 없는 HTTP 프로토콜 연결에 상태를 유지해줌. 쿠기와 비슷하지만 클라이언트에 세션 ID만 부여,
실제 데이터는 서버에 저장함

>장점 : 각 클라이언트에 고유 ID를 부여하여 정확한 구분 가능
	실제 데이터들은 서버에 저장하기 때문에 쿠키보다 보안면에서 우수함
>단점 : 데이터가 서버에 저장되므로 서버상에 처리를 요구하는 부하와 저장공간 필요
