DBMS의 특징

1. Intergrity(무결성) : 데이터 베이스 안의 데이터는 오류가 있어서는 안된다
Constraint(제약조건)을 이용해서 무결성을 보장받을수 있다

2. 데이터의 독립성 : 데이터 베이스를 사용하는 응용프로그램은 데이터 베이스의 변경에 따라 의존적이면 안된다

3. 보안 : 접근이 허용된 사람만 데이터를 사용할수 있도록 보장함

4. 중복의 최소화 : 공간(용량)의 문제나 데이터의 일관성에대한 문제를 해결함

5. 안정성 : 데이터의 백업과 복원기능을 제공함

DBMS의 발전 과정

1. 옛날에는 계층형 DBMS를 사용했음 
>> 저장은 쉬웠지만 데이터를 연관지어서 사용하기엔 힘들었음

2. 계층형 DBMS를 발전시켜서 네트워크 DBMS를 사용함

3. E.F.Codd가 Relation Data Model을 제안했는데 이 관계형 DBMS는 정형적인 데이터를 다루는데 특화됨

4. 1990년도에는 객체지향 DBMS가 등장했지만 별로 성공적이지 못했음 

5. 현대에서도 관계형 DBMS를 사용함
