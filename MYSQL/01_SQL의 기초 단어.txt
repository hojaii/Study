Table : 컬럼의 집합, 데이터를 입력하기위해 보통 표로 표현함

Database : 테이블의 저장장소이지만 테이블 이외에 다른개체도 존재함

DBMS : 데이터를 관리하는 소프트웨어의 집합

Column : 표의 열, 필드로 각 테이블은 컬럼으로 구성되게됨

Columname : 각 컬럼을 구분하기 위한 이름으로 영문식임

Data type : 열의 데이터 형식으로 숫자 문자나 날짜 형식임

Row : 표의 행, 레코드로 실질적인 데이터를 저장함

PrimaryKey : PK, 기본키라고 부름 각각의 행을 유일하게 식별할수있는 컬럼에게 부여됨 설정을 해도 안해도 상관은없지만 설정하는게좋음
	    테이블 당 한개만 존재할수있으며 지정하면 Index가 생성되고 속도가 상당히빨라짐 ex)주민등록번호, 회원ID

Index : 테이블에 컬럼단위로 설정할수있고 일반적으론 검색이 자주되는 컬럼에 생성함

View : 가상의 테이블이라고 생각하면되고 실제로는 데이터를 가지고 있지않음(SQL문장을 가지고있음) 데이터 손실을 예방하기 위해 만듬

Trigger : 일반적으로 테이블에 부록되어있고 테이블에서 insert, update, delete작업이 실행되면 실행되는 코드를 지칭함

Database Modeling : 사용하는 데이터 (도메인에서 나타나는 생성되는데이터)를  테이블안의 데이터로 변경하는 작업임
